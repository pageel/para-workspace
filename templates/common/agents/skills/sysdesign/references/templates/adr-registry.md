## Conflict Detection Guide

Before creating a new ADR, Agent MUST scan this registry for potential conflicts:

1. **Search by component**: Does an existing ADR already govern the same component/subsystem?
2. **Search by pattern**: Does an existing ADR establish a pattern that the new decision contradicts?
3. **Check superseded chain**: Is the related ADR already superseded? If so, reference the latest active ADR.

> **Convention:** ADR IDs are sequential integers (ADR-001, ADR-002, ...). The next available ID is derived from the highest existing ID + 1.
