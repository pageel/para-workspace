# v1.4.8 — Atomic Rollback, Dry-run Pipeline & README Rewrite

> **Release date:** 2026-03-04

## Highlights

- 🛡️ **Atomic Rollback** — `install.sh` now automatically restores all files if it fails mid-operation
- 🔍 **Dry-run Mode** — Preview `para install` and `para update` changes before applying
- 📄 **README Rewrite** — Installation guide redesigned with platform compatibility table and 2-step setup

## Added

- **Atomic Rollback for `install.sh`** (FEAT-30): Integrated `rollback.sh` into the install pipeline. If `install.sh` fails mid-operation, all modified files are automatically restored to their previous state. Uses `rollback_init` / `rollback_register` / `rollback_commit` lifecycle with `trap EXIT` handler.
- **`--dry-run` for `install.sh`** (FEAT-31): New `para install --dry-run` previews all file sync operations without writing any files or creating backups.
- **`--dry-run` for `update.sh`** (FEAT-31): New `para update --dry-run` skips `git pull` and passes `--dry-run` through the entire pipeline (migrate + install), enabling full update preview.
- **Windows Lock Guard**: `backup_file()` now wraps `cp` in error handling for NTFS file lock scenarios. Rollback error handler includes Windows-specific recovery hint.

## Changed

- **Kernel Version**: Bumped to `v1.4.8`.
- **CLI Help Text**: Updated `cli/para`, `install.sh`, and `update.sh` help output to document `--dry-run` option.
- **Install Header**: Mode display now shows `DRY RUN`, `FORCE`, or `Smart` explicitly.
- **Audit Logging**: `install.sh` now records `OK` or `ROLLBACK` status in `.para/audit.log` via `log_audit`.
- **README Installation Rewrite** (FEAT-33):
  - Added Prerequisites section (Git, Bash, AI Agent)
  - Added Platform Compatibility table (Antigravity ✅, Claude Code, Cursor, VS Code ⚪)
  - Simplified to 2-step install: Clone & Install → Verify
  - Unified Bash commands across Linux/macOS/Windows Git Bash
  - Added `--dry-run` and rollback documentation in Updating section

## Technical Details

### Rollback Lifecycle

```
rollback_init        → Create temp session dir
  ↓
rollback_register    → Copy original file to session (per file, before modification)
  ↓
[install operations]
  ↓
rollback_commit      → Delete session dir (success path)
  OR
trap EXIT            → rollback_execute → restore all registered files (failure path)
```

### Dry-run Pipeline

```
para update --dry-run
  ↓ skip git pull
  ↓ migrate.sh --dry-run  (preview migrations)
  ↓ install.sh --dry-run  (preview file syncs → "Would sync: ...")
  ↓ No files changed
```

### Cross-platform Safety

All CLI scripts verified clean:

- No `sed -i` (incompatible with BSD/macOS)
- No `;;&` (Bash 4+ only)
- No `readarray` / `mapfile` (Bash 4+ only)
- No `declare -A` (associative arrays, Bash 4+ only)
- `backup_file()` guards against NTFS file locks on Windows
