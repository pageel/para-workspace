# v1.4.10

## Added

- **New Workflow: `/docs`**: The official Documentation Manager workflow has been added to the governed catalog (`kernel_min: 1.4.10`).
  - **Internal-first design**: Docs are always created in `docs/` and promoted to `repo/docs/` on demand.
  - **Doc Index Pattern**: Automatically generates and maintains a lightweight `README.md` index in the `docs/` folder to save tokens and prevent directory scanning.
  - **Language Adaptation**: Automatically detects the repository's main language and translates/condenses internal native-language documents into English (or the repo's language) when publishing.
  - **4 Actions**: `new` (create doc), `review` (audit freshness), `update` (sync with code), `publish` (copy to repo).

## Changed

- **Version Bump**: Workspace version is now `1.4.10`.
- **Workflow Catalog**: `catalog.yml` updated with the new `/docs` workflow.
- **Documentation**: Updated `workflows.md` to include `/docs` in the Admin category.
- **Documentation Guide**: A new `docs.md` added under `repo/docs/workflows/` illustrating usage of the new workflow.
