# v1.5.3 — Hot Lane Refactor

> **Release Date**: 2026-03-13
> **Kernel**: 1.5.3
> **Theme**: Hot Lane, /end Sync Point, Token Optimization

---

## Summary

This release fundamentally refactors the Hybrid 3-File Architecture. The key insight: `sprint-current.md` was designed as a "Derived Focus View" rendered from backlog, but agents never actually used it that way. In practice, agents read backlog directly and sprint-current sat empty.

**v1.5.3 redefines `sprint-current.md` as a "Hot Lane"** — an agent-writable buffer for ad-hoc quick tasks that emerge during coding sessions. All synchronization is consolidated to the `/end` command via "Hot Lane Sync".

## Breaking Changes

- `/backlog update` no longer auto-syncs sprint-current.md or done.md
- `sprint-current.md` format changed from table to checklist
- Rule `hybrid-3-file-integrity.md` bumped to v2.0.0 (5 constraints replace 4)
