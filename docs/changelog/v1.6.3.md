## Summary

Transforms `project.md` into a **Central Gate** â€” the single source of truth for all context

### Path Resolution Convention

| Field        | Base path      | @ prefix resolution                              |
| :----------- | :------------- | :------------------------------------------------ |
| `strategy`   | project root   | `@{eco}/docs/x.md` â†’ `Projects/{eco}/docs/x.md`  |
| `roadmap`    | `artifacts/`   | `@{eco}/plans/x.md` â†’ `Projects/{eco}/artifacts/plans/x.md` |

### /open Agent Index Fix (Step 2.5 â†’ 2.5 + 2.6)

Step 2.5 was too long (3 sub-steps under one heading) causing agents to skip the
project-level rules/skills loading. Fix:

- **Step 2.5**: Workspace agent indices only (ALWAYS) â€” ~13 lines
- **Step 2.6**: Project agent indices (CONDITIONAL) â€” separate heading + Completion Gate

The Completion Gate requires agents to verify 4 checks before proceeding:

| # | Check                    | Required |
|:--|:-------------------------|:---------|
| 1 | Workspace rules loaded?  | ALWAYS   |
| 2 | Workspace skills loaded? | ALWAYS   |
| 3 | Project rules resolved?  | IF true  |
| 4 | Project skills resolved? | IF true  |
