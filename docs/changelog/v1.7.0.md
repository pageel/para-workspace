# v1.7.0 — Knowledge System

> **Release Date:** 2026-04-01
> **Type:** MINOR (new feature + heuristic)
> **Kernel:** H10 added (Knowledge Items)

## Summary

Introduces the Knowledge System for persistent AI agent memory across sessions.
Knowledge Items (KIs) are curated, distilled knowledge that persist in the platform's
KI Store and are automatically injected at session start.

## New Features

### Knowledge Items System

- **KI Schema** (`kernel/schema/ki.schema.json`): Graph-ready schema with domain × purpose
  2D taxonomy, `relates_to`, `code_refs`, and `concepts` fields for future Knowledge Graph.
- **`/knowledge` workflow**: Full lifecycle management — dashboard, create, update, audit,
  archive. Includes health scoring, reference validation, and auto-indexing.
- **KI Scope Loading** (`/open` Step 2.7): Field-gated conditional loading of KI index
  with scope matching (workspace/project/ecosystem).
- **KI Suggestion Hooks**: Lightweight hooks in `/end` (Step 4.7), `/retro` (Step 4.5),
  `/brainstorm` (Option F), `/learn` (Step 4.5) for proactive KI creation.
- **KI Health Check** (`/para-audit` Step 5.6): Schema compliance, reference integrity,
  staleness monitoring for all KIs.
- **KI Context in Planning** (`/plan` Step 2.7 Phase E): Pitfall KIs → Risks section,
  playbook KIs → Phase task references.

### Governance

- **H10 Knowledge Items** (11 clauses): Schema conformance, artifact requirements,
  scope rules, slug validation (`^[a-z0-9_]{3,60}$`), namespace separation (para_ prefix).
- **KR1-KR5 Knowledge Rules** (`rules/knowledge.md`): Write gate, user approval,
  namespace boundary, zone restriction, recoverability.

### Infrastructure

- **KI Index** (`.para/knowledge/index.md`): Graph-parseable trigger table for scope
  matching, auto-managed by `/knowledge` workflow.
- **KI Seed Templates** (`templates/knowledge/`): System KIs shipped with repo for
  first-install seeding via `para install`/`para update`.
- **Defense-in-Depth L2**: KI section in `/open` Step 8 safety block survives truncation.

## Bug Fixes

- **BUG-25**: Fixed `/open` grep false-positive for brainstorm pending detection.
  Changed from `grep -l "Pending"` to `grep -l "Decision.*Pending"`.
- **BUG-27**: Fixed `./para update` not removing orphan files when catalog entries
  are deleted (e.g. formatting rule → skill migration). Install now compares
  `Resources/ai-agents/` against repo templates and cleans orphans with backup.

### Migrations

- **Formatting Rule → Skill**: `formatting-tables-diagrams.md` migrated from `rules/`
  to `skills/formatting/SKILL.md`. Loaded via skills trigger index instead of rules.
  Rule catalog entry removed; skill catalog entry added.
- **Rule Frontmatter**: All 10 catalog rule files standardized with YAML frontmatter
  (`description`, `source: catalog`) for consistent orphan detection.
