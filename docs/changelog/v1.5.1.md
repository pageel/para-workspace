# Release v1.5.1 — Hybrid 3-File Synchronization

> **Date:** 2026-03-11
> **Theme:** "Fast Mode & No Amnesia"

This release finalizes the **Hybrid 3-File Model** architecture, changing how AI Agents and users interact with task management within PARA workspaces.

## 🌟 Core Features

1. **Auto-Sync `sprint-current.md` (One-way Snapshot)**
   - The `/backlog update` command now automatically scans for tasks with `🚀 ToDo` or `🔨 In Progress` statuses and renders/overwrites them into `sprint-current.md`.
   - **Benefit:** Creates an ultra-lightweight "Fast Mode" Focus View that the AI Agent can read frictionlessly daily, eliminating the need to parse massive backlog files and saving thousands of tokens.

2. **Auto-Archive `done.md` (Append-only)**
   - Added automatic completion routing: When a task is marked as `✅ Done`, it is copied and time-stamped into `done.md`.
   - Added a new **`/backlog clean`** action to perform bulk house-cleaning—sweeping all completed tasks from the backlog into the historical `done.md` bucket in one go.
   - **Benefit:** Maintains an unbloated historical log of project velocity while ensuring the canonical `backlog.md` stays lean and clean.

3. **Context Workflow Updates**
   - **`/open`**: Optimized to prioritize reading `sprint-current.md` instead of the full `backlog.md`. This "Fast Mode" cuts out 90% of token noise when opening a new session.
   - **`/plan` review**: Instead of scanning the backlog for progress, `/plan review` has been rebuilt to read directly from `done.md`. This ensures phase completion is tracked accurately and can trigger project finality automatically.
   - **`/retro`**: Re-engineered to require `done.md` as its mandatory Input Data source for completion and velocity metrics, removing the outdated reliance on scraping scattered session logs.

## 🛠 Enhancements

- **Template Improvements (`repo/templates/`):** Full audit and standardization of the "General" Profile (FEAT-25) to align perfectly with root directory structures; synchronized all workflow templates with their active v1.5.1 versions (FEAT-34).
- Kernel Version automations bumped for `project.md` and `.project.yml`.

## 🐛 Fixes & Polish

- Fixed a missing syntax keyword defect in the `new-project.md` template command structure.

---

**Upgrade Action:** Run `./para update` within your root workspace directory to sync the new engine and architecture.
