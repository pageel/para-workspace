# v1.4.9 — Centralized Backup & Workspace Cleanup

> **Release date:** 2026-03-04

## Highlights

- 🧹 **`para cleanup`** — New command to remove old backups, rollback sessions, and legacy `.bak` files
- 📦 **Centralized Backups** — `install.sh` now stores backups in `.para/backups/<date>/` instead of scattered `.bak` files
- ⚠️ **Status Warning** — `para status` alerts when cleanup is needed

## Added

- **New command: `para cleanup`** (FEAT-32): Scans and removes old backup data from the workspace.
  - `--dry-run`: Preview what would be removed without deleting
  - `--force`: Remove without confirmation prompt
  - `--keep=N`: Keep the N most recent backup days (default: 5)
  - Cleans 3 types of data:
    1. Old date-based backup folders in `.para/backups/<date>/`
    2. Committed rollback sessions in `.para/backups/rollback-*`
    3. Legacy `.bak` files scattered across the workspace (pre-v1.4.9)
  - Logs cleanup actions to `.para/audit.log`

- **Status cleanup warning**: `para status` now detects legacy `.bak` files and old rollback sessions, showing a `⚠️` hint to run `./para cleanup`.

## Changed

- **Centralized backup storage**: `backup_file()` in `install.sh` now saves backups to `.para/backups/<date>/` with flattened filenames (e.g., `.agent_workflows_plan.md`) instead of creating `.bak` files next to the original. This keeps the workspace tree clean.
- **CLI help text**: Added `cleanup` command with options to `para --help`.
- **Kernel Version**: Bumped to `v1.4.9`.
