# v1.5.4 — Defense-in-Depth: Context Recovery & Rule Guards

> **Release Date**: 2026-03-17
> **Kernel**: 1.5.4
> **Theme**: Context Recovery, Rule Priority, Workflow Pre-flight, File Guard Headers

---

## Summary

This release addresses a critical gap: after **context truncation** (checkpoint, sliding window), agents lose all rules loaded at session start and may silently violate constraints. v1.5.4 implements a **4-layer Defense-in-Depth** strategy where each layer covers the gaps of the previous one.

The design was validated by a real-world dogfooding incident: the agent violated `done.md` append-only constraint (C2) during this very session, which led to discovering the "workflow bypass" gap and creating Layer 4 (File Guard Headers).

## 4-Layer Architecture

```
Layer 1: Rule file (agent-behavior.md §4)
  └─ Context Recovery protocol + File-Level Guards table
  └─ Covers: workflow bypass (direct edits)
  └─ Type: Passive (depends on agent detecting decay)

Layer 2: Workflow output (/open Safety Block)
  └─ Compact rules reminder in Step 8 report
  └─ Covers: checkpoint truncation (survives in summary)
  └─ Type: Passive (informational)

Layer 3: Workflow pre-flight (Step 0)
  └─ 7 workflows re-read rules.md from disk
  └─ Covers: normal workflow usage after truncation
  └─ Type: Active (force re-read)

Layer 4: File Guard Headers (C6) ⭐
  └─ Inline <!-- ⚠️ --> in done.md, sprint-current.md
  └─ Covers: all remaining cases (Proximity Principle)
  └─ Type: Active (agent sees guard before editing)
```

## Token Impact

```
Defense-in-Depth total cost: ~200-275 tokens/session
  L1: ~100 tokens (on-demand rule re-read)
  L2: ~40 tokens  (Safety Block in /open report)
  L3: ~60-105 tokens (Step 0 × 4-7 workflows/session)
  L4: ~0-30 tokens (guard headers per file read)

Overhead: 3-5% of session budget — negligible for comprehensive protection.
```

## Dogfooding Validation

The agent violated C2 (wrote directly to `done.md`) during this session while implementing FEAT-47. Root cause analysis:

1. **Context decay** — long conversation, rules lost after checkpoint
2. **Workflow bypass** — agent edited files directly instead of using `/backlog`
3. **"Helpful bias"** — agent tried to complete multiple actions at once

This incident exposed Layer 3's gap (pre-flight only works if workflow is used) and directly led to creating Layer 4 (File Guard Headers) and the File-Level Guards table in Layer 1.
