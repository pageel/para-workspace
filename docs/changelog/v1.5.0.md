# v1.5.0

## Added

- **New Workflow: `/update`**: Agent-guided safe workspace update with pre-flight checks and error recovery (`kernel_min: 1.5.0`).
  - **Pre-flight checks**: Validates network connectivity, git status, and version state before updating.
  - **Dry-run preview**: Runs `para update --dry-run` first so users can review changes before applying.
  - **Error recovery**: Built-in diagnosis table for common failures (network, git conflicts, permissions, rollback).
  - **Post-update verification**: Confirms version, migration history, and audit log after update.

- **New Workflow: `/brainstorm`** (Governed): Collaborative ideation with structured output and 5 exit paths (`kernel_min: 1.5.0`).
  - Saves structured analysis to `artifacts/para-decisions/brainstorm-[topic]-[date].md`.
  - **5 exit paths**: Seeds (incubate), Plan (formalize), Backlog (simple tasks), Doc (preserve as reference), Learn (extract reusable knowledge).
  - Integrated with `/plan` Step 2.5 — plan auto-discovers recent brainstorm outputs.

- **Project Rules Loading (Progressive Disclosure)**: New Rule #4 in `context-rules.md`.
  - Agent checks for a lightweight `rules.md` index at `Projects/<project>/.agent/rules.md`.
  - Rules are loaded **on demand** — only when the current action matches a trigger, not upfront.
  - New template `rules.md` included in project scaffolding.
  - Workflow `/open` updated with Step 2.5 to auto-load project rules index.
  - **Kernel heuristic H2** updated with the lazy-loading protocol definition.

## Fixed

- **`update.sh` skipping migrations**: `CURRENT_VER` was read from `repo/VERSION` (already updated by `git pull`), causing it to always equal `NEW_VER` and skip `migrate.sh`. Now reads from `.para-workspace.yml` (the workspace's actual installed version).
  - **Impact**: `history.log` was not recording updates from v1.4.7 onwards.

## Changed

- **Version Bump**: Workspace version is now `1.5.0`.
- **Workflow Catalog**: `catalog.yml` updated with `/update` and `/brainstorm` workflows.
- **Workflow `/open`**: Added Step 2.5 for project rules index loading. Version bumped to 1.5.0.
- **Workflow `/plan`**: Added Step 2.5 for brainstorm context discovery, renumbered Learnings to Step 2.6. Added `/brainstorm` to Related. Version bumped to 1.5.0.
- **Workflow `/para-rule`**: Added step to update `rules.md` index when adding project-specific rules; updated Context Routing section. Version bumped to 1.5.0.
- **Documentation**: Added `docs/workflows/update.md`, `docs/workflows/open.md`, `docs/workflows/brainstorm.md`, `docs/workflows/plan.md`, `docs/project-rules.md`; updated `docs/workflows.md` and both READMEs.
