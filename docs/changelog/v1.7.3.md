# v1.7.3 — Agent Path Convention Fix (BUG-28)

> **Released:** 2026-04-02
> **Previous:** v1.7.2

## Summary

Renamed `.agent/` directory to `.agents/` across the entire codebase to match
Antigravity platform's default convention. Added Antigravity-compatible frontmatter
(`trigger`, `glob`, `description`) to all governed rules.

## Breaking Changes

None — `para update` includes automatic migration from `.agent/` to `.agents/`.

## Changes

### Bug Fixes

- **BUG-28:** Fixed `.agent/` vs `.agents/` path mismatch that caused Antigravity UI
  to display rules from a different directory than where PARA workspace stores them.
  All 68 files in the repo have been updated to use `.agents/` consistently.

### Improvements

- **Rule Frontmatter:** All 12 governed rules now include Antigravity-compatible
  frontmatter with `trigger`, `glob`, and improved `description` fields.
  This enables better integration with Antigravity's rule management UI.

### Migration

- **Automatic:** `para update` (via `migrate.sh`) will auto-rename `.agent/` to
  `.agents/` for both workspace root and all project directories.
- **Merge logic:** If both `.agent/` and `.agents/` exist (e.g., user created rules
  via Antigravity UI), contents are merged safely before removing the old directory.
