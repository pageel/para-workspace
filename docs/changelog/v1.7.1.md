## New Features

- **`/knowledge system [topic]`** — Create/update system KIs with version alignment and template baseline
- **`/knowledge system update`** — Sync system KIs from repo templates (merge-safe: template wins metadata, union merge references)
- **`/knowledge system defaults`** — Initialize all default system KIs from templates
- **Namespace guard** — Reject `para_*` prefix for user KIs (enforced by KR3)
- **KR6** — System KI Governed Lifecycle rule: template source, update sync, merge strategy
- **Dashboard split** — System KIs vs User KIs sections in `/knowledge` dashboard

## Migration

No migration needed. `./para update` handles all changes automatically. System KIs are auto-synced when template `para_version` changes.

## Breaking Changes

None. All changes are backward-compatible.
