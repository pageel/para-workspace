# v1.6.4 — Para-Kit Skill v1.1.0, Recursive Sync & Git Hash Detection

> **Released**: 2026-03-30 | **Kernel**: 1.6.4

## Summary

This release restructures the `para-kit` skill into a lean structure reference (v1.1.0), replaces the 1-level directory sync with a fully recursive `sync_directory_recursive()`, and introduces git commit hash-based change detection in `update.sh` to handle hotfixes without version bumps.

## Features

### FEAT-55: Para-Kit Skill Rewrite (v1.1.0)

- **SKILL.md** rewritten as a 5-section structure reference:
  1. PARA Workspace Structure (4 pillars + project layout)
  2. `project.md` Schema (all v1.6.3 fields including `upstream`)
  3. Quick Reference Card — Kernel Governance (I1-I11 + H1-H9 one-liners)
  4. Selection Strategy (CLI vs Workflow routing)
  5. On-demand References (templates + examples)
- **Colocation**: `templates/` and `examples/` now live inside the skill folder for automatic sync
- **Generic examples**: `project-schema-vectors.md` uses fictional project names (acme-platform, my-app)
- **Removed**: `formatting-tables-diagrams.md` duplicate from skill templates (now a governed rule)

### FEAT-56: Recursive Directory Sync in install.sh

- New `sync_directory_recursive()` helper handles arbitrarily nested subdirectories
- Replaces the 1-level `for` loop that could not sync `skills/para-kit/templates/` or `examples/`
- Syncs **all file types** (not just `.md`) — supports YAML, JSON, etc.
- Uses `|| true` to prevent `set -e` crashes per `bash-cli-gotchas.md` learnings

## Bug Fixes

### BUG-19: Formatting Rule Not Triggered

- **Root cause**: `formatting-tables-diagrams.md` existed only in the workspace (from a dogfooding experiment) without an entry in `rules.md` or `catalog.yml` → agent never triggered it
- **Fix**: Copied rule to repo, added YAML frontmatter, added entry to `rules.md` index and `catalog.yml`

### BUG-20: update.sh Version Comparison Logic

- **Root cause**: `CURRENT_VER` fell back to `repo/VERSION` which would read the *same file* before and after `git pull`, making versions always equal → migration skipped
- **Fix**: `CURRENT_VER` now reads exclusively from `.para-workspace.yml` (workspace copy), with `0.0.0` fallback for fresh installs

### BUG-21: Hotfixes Not Detected by update.sh

- **Root cause**: Version-only comparison could not detect hotfix commits that don't bump `VERSION`
- **Fix**: Git commit hash comparison (`OLD_COMMIT` vs `NEW_COMMIT`) before/after `git pull`. Version comparison now only gates migration, not sync. `install.sh` ALWAYS runs (idempotent).
