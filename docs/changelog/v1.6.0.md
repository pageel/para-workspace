# v1.6.0 — Meta-Project & Ecosystem Support

> **Release Date**: 2026-03-20
> **Kernel**: 1.6.0
> **Theme**: Multi-project coordination, cross-project plans, ecosystem governance
> **RFC**: [RFC-0003](../../rfcs/0003-meta-project-governance.md)

---

## Summary

This release introduces **Ecosystem Projects** — a meta-project concept that enables multi-project coordination within the standard PARA structure. An ecosystem project (`type: ecosystem`) acts as a coordinator for related satellite projects, holding shared strategy, cross-project plans, and roadmaps — but no source code.

Key capabilities:
- **Project types**: `standard` (default) or `ecosystem` (meta-project, no `repo/`)
- **Satellite linking**: bidirectional references via `ecosystem` and `satellites` fields
- **@prefix resolution**: `@my-ecosystem/plans/roadmap.md` resolves across project boundaries
- **Workflow adaptation**: 5 workflows detect ecosystem type and behave accordingly
- **Consistency validation**: `/para-audit` validates ecosystem ↔ satellite integrity

## Breaking Changes

None. All new fields are optional with safe defaults. Existing projects work without changes.

## Architecture

### Ecosystem Model

```text
┌───────────────────────────────────────────────────────┐
│                       Projects/                       │
│                                                       │
│  ┌────────────────────────────┐                       │
│  │     my-ecosystem/          │  ECOSYSTEM            │
│  │  ├─ project.md             │  type: ecosystem      │
│  │  ├─ artifacts/plans/  ─────┼── Shared plans        │
│  │  └─ docs/strategy.md       │  No repo/             │
│  └─────────────┬──────────────┘                       │
│                │  @my-ecosystem/ prefix                │
│      ┌─────────┼─────────┬──────────┐                 │
│      ▼         ▼         ▼          ▼                 │
│  ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐             │
│  │ app-a │ │ app-b │ │ lib-c │ │ lib-d │ SATELLITES  │
│  └───────┘ └───────┘ └───────┘ └───────┘             │
└───────────────────────────────────────────────────────┘
```

### @Prefix Resolution

```
Input:   @my-ecosystem/plans/roadmap.md
Output:  Projects/my-ecosystem/artifacts/plans/roadmap.md

Rule:    @{name}/path → Projects/{name}/artifacts/path
Access:  Read-only (satellites cannot modify ecosystem files)
```

### Workflow Decision Flow

```text
                   /open [project]
                        │
                  ┌─────┴──────┐
                  │ project.md │
                  └─────┬──────┘
                        │
                 type = ecosystem?
                 ┌──────┴──────┐
                YES            NO
                 │              │
          ┌──────┴──────┐  ┌───┴──────────┐
          │  Skip git   │  │ Normal git   │
          │  List sats  │  │ flow         │
          └─────────────┘  └───┬──────────┘
                               │
                        has ecosystem?
                        ┌──────┴──────┐
                       YES            NO
                        │              │
                 ┌──────┴──────┐  ┌───┴───────┐
                 │ Resolve @   │  │  Normal   │
                 └────────────┘  └───────────┘
```

## Migration

No migration required. All new fields are optional:
- `type` defaults to `"standard"`
- `ecosystem` defaults to `null`
- `satellites` defaults to `null`
- `active_plan` continues to work without `@` prefix

Users opt-in to ecosystem features by adding `type: ecosystem` to `project.md`.
