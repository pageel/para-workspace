## 🌌 Overview

Version **v1.9.4** optimizes the management and synchronization of AI Agents (Workflows, Rules, Skills) in the PARA Workspace CLI via the `./para install-tool --sync` command. This version completely eliminates network latency during local development, ensures data integrity through source manifest overwriting, and automates catalog index registration to ensure AI Agents load optimal context.

Additionally, this release officially packages and releases all **18 staged template updates** from previous iterations (including smart plan link pointers, platform harness safety gates, Vibecode session control, and advanced HTML Renderer layout optimizations).

---

## 🛠️ Key Changes

### 1. Local Development Sync
- **DX Optimization:** The `./para install-tool <tool> --sync` command automatically checks if a local project directory exists under `Projects/<tool>/repo/templates/agents` or `Projects/para-<tool>/repo/templates/agents`.
- **Direct Copying:** If a local directory is detected, the installer directly copies all templates and the manifest file (`tool.manifest.yml`) from the local project to the global installation path, bypassing remote GitHub API calls. This enables immediate testing of agent changes without needing to commit/push to GitHub.

### 2. Source Manifest Overwrite
- The installer automatically copies or downloads the latest `tool.manifest.yml` from the development source (local dev path or raw GitHub URL) and overwrites the stale manifest in the system directory `.para/tools/<tool>/tool.manifest.yml`.
- This ensures that subsequent rule/skill scanning is always based on the latest configuration.

### 3. AI Index Auto-Sync
- Automatically parses rule/skill metadata (`name`, `trigger`, and `priority`) defined in `tool.manifest.yml`.
- Dynamically registers or updates corresponding index rows in `.agents/rules.md` (for rules) and `.agents/skills.md` (for skills) at the workspace root.
- Prevents duplicate rows by scanning relative paths (e.g., `rules/my-rule.md`) and updating fields in-place if they already exist.

### 4. Cross-Platform Compatibility & Security
- **Windows Carriage Returns:** Strips `\r` (CRLF) characters when parsing YAML streams line-by-line, preventing path resolution failures on Windows.
- **Windows find.exe Conflict:** Dynamically resolves the POSIX `find` binary path (at `/usr/bin/find` or `/bin/find`) to prevent execution conflicts with the native Windows `find.exe` utility.
- **Sed Replacements Escaping:** Automatically escapes `/` to `\/` and `&` to `\&` in replacement variables within `sed` commands, preventing shell substitution syntax failures and Command Injection.
- **Path Traversal Protection:** Implements validation filters that block relative paths containing `..` in both source and target variables within `tool.manifest.yml` to protect against unauthorized file writes.
